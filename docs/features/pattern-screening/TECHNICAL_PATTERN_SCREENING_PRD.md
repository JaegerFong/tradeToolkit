# 技术形态选股（老鸭头 / N 字形态）PRD

版本：v0.1（首期）

## 1. 背景

当前系统具备 A 股本地数据（MongoDB 的 `stock_daily_quotes`/`market_quotes`/`stock_basic_info` 等）、前端筛选/任务中心/股票详情页，但缺少“技术形态”类选股能力。希望新增“技术形态选股”功能，首期支持两种经典形态：**老鸭头**与**N 字形态**，并以后台任务形式执行，可查看过程，输出结构化结果并可下钻到个股入选分析与交易计划。

## 2. 目标与非目标

### 2.1 目标

- 支持两种形态：**老鸭头**、**N 字形态**
- **仅使用本地已下载 A 股数据**完成全市场扫描（不依赖在线行情接口）
- 前端提供可操作界面：选择形态、参数、股票池范围、提交后台任务、查看结果与过程
- 任务过程可追溯：展示步骤、进度、关键依据摘要（不展示模型隐藏推理）
- 结构化输出必须包含：
  - 列表：编码、名称、价格、当天涨跌额、涨跌幅、市值、形态、推荐值
  - 详情：入选分析、形态拆解、后续走势预期、推荐值、买入价区间、仓位、止损、风险点、失效条件

### 2.2 非目标（首期不做）

- 自动下单/实盘交易
- 分钟级形态扫描
- 港股/美股形态扫描
- 用户自定义形态 DSL
- 盘中实时连续扫描推送
- 展示完整 chain-of-thought

## 3. 用户故事

- 作为用户，我希望在“技术形态选股”页面选择“老鸭头”，设置参数后点击开始，系统在后台扫描全 A 股并返回入选股票列表。
- 作为用户，我希望在任务中心看到选股任务的实时进度、当前阶段，以及关键判断依据的摘要。
- 作为用户，我希望点击某只入选股票后看到：它的形态拆解、为什么入选、后续走势预期、买入价、仓位与止损建议。

## 4. 关键交互

### 4.1 页面入口

- 新增菜单：**技术形态选股**
- 路由：
  - `/pattern-screening`：创建任务/查看最近任务
  - `/pattern-screening/tasks/:taskId`：任务详情（过程 + 结果 + 下钻）

### 4.2 任务流程

1. 用户提交任务（形态、参数、股票池、时间窗口、是否启用 LLM 复核）
2. 后端创建任务并异步执行
3. 前端轮询或订阅任务进度，展示“过程时间线”
4. 任务完成后展示结构化结果列表
5. 点击股票打开“入选详情”（可用抽屉或新页面）

## 5. 形态定义（业务描述）

### 5.1 老鸭头（Duck Head）

拆解四段：

- **鸭颈**：前期上升段，均线向上，价格在关键均线上方运行
- **鸭头**：上涨后回调/整理，波动收敛，成交量缩小
- **鸭鼻**：短均线重新拐头，价格逼近整理上沿，量能恢复
- **张嘴**：放量突破整理平台/前高，重新进入上升

首期规则检测输出必须包含：各段起止日期、区间涨跌、回撤、量能对比、突破条件是否满足。

### 5.2 N 字形态（N-Shape）

拆解三段：

- **第一笔上升**：低点到高点
- **中段回调**：高点回落但不破坏趋势结构
- **第二笔上升**：再次启动，接近或突破前高/关键位

首期规则检测输出必须包含：三段时间范围、涨跌幅、回调深度、第二段启动信号与风险点。

## 6. 结果展示与字段（产品口径）

### 6.1 列表字段（必含）

- 股票编码 `code`
- 股票名称 `name`
- 最新价 `price`
- 当天涨跌额 `change_amount`
- 当天涨跌幅 `pct_chg`
- 市值 `market_cap`
- 形态类型 `pattern_type`（`laoyatou`/`n_shape`）
- 形态评分 `pattern_score`（0-100）
- 推荐值 `recommendation_score`（0-100）
- 简要原因 `brief_reason`

### 6.2 详情字段（必含）

- 形态拆解 `pattern_breakdown`（老鸭头四段 / N 字三段）
- 入选分析 `analysis`（面向用户的可审计解释摘要）
- 后续走势预期 `trend_expectation`
- 买入价格区间 `buy_price_range`
- 仓位建议 `position_suggestion`
- 止损位 `stop_loss`
- 风险点 `risk_points`
- 失效条件 `invalid_conditions`

## 7. 合规与免责声明

- 页面与结果必须包含“研究与教育用途”风险提示
- 不展示 LLM 隐藏推理链，只展示可审计的依据摘要与结构化结论
